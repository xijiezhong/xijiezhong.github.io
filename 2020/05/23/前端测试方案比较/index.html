前端测试方案比较

2020-05-23

对于前端来讲，可选的测试方案很多，为了选择适合自己项目的测试工具及方案，首先需要对现有的各大工具有所了解。下面将对目前的测试方案及工具做一些介绍和比较。

测试方案

前端的测试主要包含三大块：代码上的规范、单元测试以及集成测试。

代码规范

代码规范主要通过一些检测代码的工具和同事的code review来保证。

前端的代码规范工具主要包括：Eslint、Standard、Prettier、JSHint。

  • JSHint是由古老的JSLint发展而来，JSLint不可配置、不可扩展，而JSHint其实就是可配置的JSLint版本。

  • Prettier在社区很火，具有可配置化、多语言支持、配置项简洁的优点。

  • ESLint，易于扩展，可自定义规则，可以插件形式安装更多的规则

单元测试

单元测试的工具包括：Jasmine/Jest/Mocha/AVA等

  • Jasmine：开箱即用、非常成熟、支持断言和仿真

  • Jest：配置少，API简单，基于Jasmine，来自Facebook，比较新颖

  • Mocha：灵活、社区成熟、需要较多配置

  • AVA：异步，性能好、简约、清晰快照测试断言需要第三方支持

集成测试

对于前端工程来讲，集成测试主要有三大目的

  1. 保证UI：确保浏览器渲染正常，实现快照对比。

  2. 克服兼容：在集成测试端，快速进行多种浏览器的测试，发现兼容问题

  3. 易于回归：易于流程控制，用户交互模拟，方便测试用例编写

前端E2E的测试框架主要包括：Cypress/NightWatch/Puppeteer等

  • Cypress：不使用Selenium，运行速度要快，比较新颖，问题较多

  • NightWatch：上手较快、社区成熟、API丰富

  • Puppeteer: 基于headless chrome,支持测试Chrome插件，可控性大，只支持chrome

Vue项目的单元测试实践

这里，我选择了mocha+chai来搭建项目的单元测试。

chai是一套TDD(测试驱动开发)/BDD(行为驱动开发)的断言库。包含有3个断言库支持BDD风格的expect/should和TDD风格的assert。

对于Vue项目来讲，引入mocha、chai分两种情况

新项目

如果你还没有创建项目，那么你可以通过vue-cli迅速的创建好一个配置好mocha+chai的项目

步骤如下

  1. 运行 vue create xxx命令（xxx为项目名）
  2. 在预配置中勾选Unit Testing
  3. 然后在选择测试方案中选择mocha + chai
  4. 确定并等待安装完毕

你会看到项目中已经配置好了mocha+chai并建好了一个模板test文件。npm install装好依赖过后运行npm run test:unit命令就可以运行单元测试了

老项目

如果你的已有项目在创建时没有选择unit testing的预配置，那么你可以通过运行vue add @vue/unit-mocha命令来配置好mocha+chai。
